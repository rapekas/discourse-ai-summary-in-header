# Context for AI / future sessions

- **Repo:** `discourse-ai-summary-in-header` — Discourse **theme component** that clones the discourse-ai summarize button from the topic map into `.timeline-controls` (topic timeline sidebar). Does not fork discourse-ai.
- **JS:** `javascripts/discourse/lib/relocate-ai-summary.js` (`cloneButtonToTimeline`) — clones `.topic-map .ai-summarization-button` into `.timeline-controls`, wires proxy click to the original Glimmer-rendered button. `javascripts/discourse/initializers/move-ai-summary-to-header.js` — debounced `MutationObserver` on `#main-outlet`, `api.onPageChange`, applies `ai-summary-keep-in-reader-mode` class on `<html>`.
- **Why clone + proxy:** The AI summarize button is a **Glimmer component** (`AiSummaryTrigger` → `DButton` with `@action`). `cloneNode(true)` does **not** copy `addEventListener` handlers. We clone for display and proxy clicks back to the original button via `addEventListener("click", () => originalBtn.click())`.
- **SCSS:** `common/common.scss` — button height (`39px`), `gap: 2px` on `.timeline-controls`, Reader Mode compatibility overrides. Use **`@use "lib/viewport"`** and **`@include viewport.until(md)`**, not `@include breakpoint(...)`.
- **Reader Mode compatibility:** The `discourse/reader-mode` theme component ([github.com/discourse/reader-mode](https://github.com/discourse/reader-mode)) hides timeline buttons via `.reader-mode .timeline-controls .btn:not(.reader-mode-toggle) { opacity: 0; }`. Our `keep_in_reader_mode` setting overrides this for `.ai-summary-timeline-btn` and `.toggle-admin-menu` (opacity 0.4, hover 0.7). Reader Mode also adds `margin-right: 0.5em` on `.reader-mode-toggle` — we override with `button.reader-mode-toggle { margin-right: 0 }` (element + class for higher specificity, no `!important`).
- **Settings:** `settings.yml` defines `show_in_timeline` (bool, default `true`) and `keep_in_reader_mode` (bool, default `false`). Accessed in JS as `settings.show_in_timeline` (global, no import needed).
- **Tests:** QUnit `test/javascripts/unit/relocate-ai-summary-test.js` (tests `cloneButtonToTimeline`); system `spec/system/topic_page_spec.rb`. QUnit imports use `discourse/theme/lib/<filename>` namespace (not relative paths — they break in CI). System specs fabricate a post and visit with explicit `post_number`.
- **Core features system spec:** `spec/system/core_features_spec.rb` follows **Horizon** pattern. Ruby requires **quoted symbols**: `:"topics:create"` (not `:topics:create`).
- **Locales:** `locales/en.yml` and `locales/ru.yml` — setting descriptions under `theme_metadata.settings`.
- **CI:** `.github/workflows/discourse-theme.yml` reuses `discourse/.github` workflow; lint job uses `discourse/discourse_test:slim`, system/frontend tests use `slim-browsers`.
- **Cursor:** `.cursor/rules/` — local-only rules (in `.gitignore`). After edits, suggest a **Commit message** in chat (Conventional Commits).
