# Context for AI / future sessions

- **Repo:** `discourse-ai-summary-in-header` — Discourse **theme component** that adds the discourse-ai summarize button to `.timeline-controls` (topic timeline sidebar), `.d-toc-main` (Table of Contents sidebar), or `#topic-title .title-wrapper` (mobile / narrow desktop fallback). Does not fork discourse-ai.
- **JS:** `javascripts/discourse/lib/relocate-ai-summary.js` — shared `createSummaryButton()`, exported `addButtonToTimeline()`, `addButtonToToc()`, `addButtonToTitle()`, and `removeStaleButtons()`. Creates own button, dynamic imports `AiSummaryModal`, uses modal service. `removeStaleButtons(keepClass, doc)` removes buttons from non-current locations on layout transitions (e.g. narrow → wide resize). `javascripts/discourse/initializers/move-ai-summary-to-header.js` — `MutationObserver` on `#main-outlet` and `document.body`, `api.onPageChange`, applies `ai-summary-keep-in-reader-mode` class on `<html>`. On mobile (`site.mobileView`) or narrow desktop (no `.timeline-controls` / `.d-toc-main` in DOM), places button in `#topic-title .title-wrapper`.
- **Why create own button + direct modal call:** The AI summarize button is a **Glimmer component** (`AiSummaryTrigger` → `DButton` with `@action`). `cloneNode(true)` does **not** copy `addEventListener` handlers. We create our own button, load `AiSummaryModal` via `requirejs()`, and use the Discourse `modal` service to open it directly. This avoids dependency on the original button's presence in the DOM.
- **requirejs vs import():** Discourse plugin modules are AMD (`requirejs.entries`). Dynamic `import()` does **not** resolve them — use `requirejs(moduleName)` instead. The modal module path is `discourse/plugins/discourse-ai/discourse/components/modal/ai-summary-modal`.
- **SCSS:** `common/common.scss` — timeline button height (`39px`), `gap: 2px` on `.timeline-controls`, ToC button full-width + margin, Reader Mode compatibility overrides. Use **`@use "lib/viewport"`** and **`@include viewport.until(md)`**, not `@include breakpoint(...)`.
- **Reader Mode compatibility:** The `discourse/reader-mode` theme component hides timeline buttons via `opacity: 0`. Our `keep_in_reader_mode` setting overrides this for `.ai-summary-timeline-btn` and `.toggle-admin-menu` (opacity 0.4, hover 0.7). Reader Mode also adds `margin-right: 0.5em` on `.reader-mode-toggle` — we override with `button.reader-mode-toggle { margin-right: 0 }` (higher specificity, no `!important`).
- **Table of Contents integration:** When `show_in_toc` is enabled, `addButtonToToc()` inserts a button at the top of `.d-toc-main` (from the [DiscoTOC](https://github.com/discourse/DiscoTOC) theme component). Uses the same `createSummaryButton()` with `ai-summary-toc-btn` class.
- **Settings:** `settings.yml` defines `show_in_timeline` (bool, default `true`), `show_on_mobile` (bool, default `true`), and `keep_in_reader_mode` (bool, default `false`). On desktop: when `show_in_timeline` is on — ToC → timeline → title-wrapper fallback (narrow screens). On mobile: when `show_on_mobile` is on, button appears in topic title area. Accessed in JS as `settings.<name>` (global, no import needed).
- **Tests:** system specs only — `spec/system/topic_page_spec.rb` and `spec/system/core_features_spec.rb` (shared example from Discourse core, **Horizon** pattern). No QUnit/frontend tests (removed — they timed out in CI because `themes:qunit` runs in full Rails and our pure-DOM unit tests didn't load). System specs fabricate a post and visit with explicit `post_number`. Ruby requires **quoted symbols**: `:"topics:create"` (not `:topics:create`).
- **Locales:** `locales/en.yml` and `locales/ru.yml` — setting descriptions under `theme_metadata.settings`.
- **CI:** `.github/workflows/discourse-theme.yml` reuses `discourse/.github` workflow. Jobs: `linting` (slim), `backend_tests`, `system_tests` (slim-browsers). No `frontend_tests` — the reusable workflow auto-detects test types by checking `test/**/*.{js,gjs}` presence; without that dir, the frontend job is skipped (same as [DiscoTOC](https://github.com/discourse/DiscoTOC)).
- **Cursor:** `.cursor/rules/` — local-only rules (in `.gitignore`). After edits, suggest a **Commit message** in chat (Conventional Commits).
